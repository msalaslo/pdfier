PDFier offers a developers Application Programming Interface (API) for URL webpage or HTML to PDF/UA conversion. It lets you create PDF/UA documents quickly, efficiently and cost-effectively. Use it to create high-quality invoices, reports, tickets, e-books, brochures and much more!

Features

  • Supports full HTML, including HTML5, CSS2 and some features of CSS3.
  • Ready to use code examples for Java, .NET / C#, PHP, Perl, Ruby, Python and VB.net!
  • No installation required, just invoke our online services

URL To PDF/UA Usage

The PDFier API is easy to use. It takes a license and either a URL or raw HTML as input and returns the PDF/UA.
You just send a request similar to this, which converts http://www.example.com to PDF/UA: 

http://pdfier.com/pdfier-app/urltopdfua?license=yourlicensekey&url=http://www.example.com

The request should at least have the following components:

  1. endpoint: this should be http://pdfier.com/pdfier-app/urltopdfua or https://pdfier.com/pdfier-app/urltopdfua for secure access
  2. license: this is mandatory and you will get yours when you sign up
  3. url: the web page that you want to convert OR

Other than that you can use a lot of additional parameters in the API call, which give you access to all the features for conversion. These are all specified below.

NB: You can use both GET as well as POST requests as long as you properly URL encode the data that you pass.

HTML To PDF/UA Usage

The PDFier API is easy to use. It takes a license and either a URL or raw HTML as input and returns the PDF/UA.
You just send a request similar to this, which converts http://www.example.com to PDF/UA: 

http://pdfier.com/pdfier-app/htmltopdfua?license=yourlicensekey&html=%3Cdiv%20class%3D%27main-content%27...%3C%2Fdiv%3E

The request should at least have the following components:

  1. endpoint: this should be http://pdfier.com/pdfier-app/htmltopdfua or https://pdfier.com/pdfier-app/htmltopdfua for secure access
  2. license: this is mandatory and you will get yours when you sign up
  3. html: the raw HTML code that you want to convert. HTML must be URL encoded. HTML encode the data that you pass.
  4. language: Set the language of the PDF/UA instead of the value of the <lang> attibute from the web page.
  5. title: Set the title of the PDF/UA instead of the value of the <title> tag from the web page.

Other than that you can use a lot of additional parameters in the API call, which give you access to all the features for conversion. These are all specified below.

NB: You should use POST requests, HTML could be too long for GET request usage. For more information check GET method Restrictions on data length

Conversion Parameters

The table below shows the settings that you can change to control the conversion process. These will largely determine the contents of the resulting PDF/UA.

Parameter Description
language Set the language of the PDF/UA instead of the value of the <lang> attibute from the web page. Mandatory when use HTML to PDF/UA.
title Set the title of the PDF/UA instead of the value of the <title> tag from the web page. Mandatory when use HTML to PDF/UA.
filename Set the filename of the PDF/UA, which you see in the top bar when you open the PDF/UA in Adobe Reader.

Return codes

Our API returns HTTP response codes, which you can check to see if the conversion was succesfull or not. The following is the list of return codes we use.

Code Description
200 OK Your conversion was processed succesfully
400 Bad Request Some request params are missing or bad specified
402 Payment required You specified an invalid license key
429 Too Many Requests You have overrun a usage limit for your plan
503 Service unavailable You are sending multiple requests at the same time to the API from the same IP address
